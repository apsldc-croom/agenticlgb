# Commit Strategy

Commit message and frequency.

## Commit Format

```
{type}({scope}): {subject}

{body}

{footer}
```

### Types

| Type | Description |
|------|-------------|
| feat | New feature |
| fix | Bug fix |
| refactor | Code refactor |
| test | Tests |
| docs | Documentation |
| chore | Maintenance |

## Example Commits

```
feat(auth): add JWT refresh token

- Implement refresh token endpoint
- Add token expiry validation
- Update auth middleware

Closes #123
```

```
fix(api): resolve serialization error

- Handle None values in serializer
- Add default values for optional fields
```

## Auto-Commit Rules

```yaml
commit:
  auto: true
  min_changes: 1
  max_per_task: 10
  
  message_template: |
    {type}({scope}): {summary}
    
    Task: {task_id}
    Generated by: {agent}
```

## Validation

```python
def validate_commit(message):
    if len(message) > 72:
        return False, "Subject too long"
    if not has_type(message):
        return False, "Missing type"
    return True, "OK"
```
